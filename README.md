# Task-14-cyber-security-internship
Linux-Server-Hardening-Secure-Configuration

##  Overview

This repository contains my submission for **Task 14: Linux Server Hardening & Secure Configuration**.

The goal of this task was to secure a Linux server by reducing vulnerabilities and applying basic hardening techniques.

---

## What I Performed

* Reviewed system users and running services
* Removed unused accounts
* Restricted sudo access
* Disabled root login via SSH
* Enabled SSH key authentication
* Updated system packages
* Enabled automatic security updates
* Configured UFW firewall
* Disabled unnecessary services
* Secured file permissions
* Reviewed authentication logs

---

## Key Learning

* Understanding server hardening
* Applying least privilege principle
* Reducing attack surface
* Improving system security monitoring

---

## Outcome

The Linux server is more secure and protected against common attacks like brute-force and unauthorized access.
